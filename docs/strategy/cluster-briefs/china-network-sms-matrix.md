# Brief：中国网络与短信实测矩阵

- 建议 URL：`/research/china-network-sms-matrix/`
- 主关键词：`giffgaff 中国网络短信实测`
- 意图：查看“某城市、设备、当地网络、SIM 形态在某日的可观察结果”，不是寻求永久覆盖承诺。
- 发布门槛：至少有真实原始记录和公开方法；只有空表或待测项时不索引发布。

## 最小数据模型

每条测试记录必须有：

- 匿名 `testId` 与测试日期/时区。
- 城市级位置，不采集精确地址。
- 手机精确型号、地区版本、OS 版本。
- SIM 形态（实体/eSIM）及是否已知账号正常。
- 自动/手动选网、显示的当地网络、信号状态。
- 普通入站 SMS、官方 SMS、主动 SMS、通话、数据分别的测试结果。
- 重试次数、等待窗口、错误文案、失败记录。
- 测试者类型（编辑实测/可验证投稿）与证据等级。

禁止保存号码、IMSI/ICCID 全值、账号、密码、验证码、Cookie、二维码或支付信息。

## 方法

1. 先验证账号和 SIM 状态。
2. 固定设备与环境，一次只改变一个变量。
3. 记录自动选网结果，再按官方说明做手动选网。
4. 普通短信与平台 OTP 分开；本矩阵只回答运营商层。
5. 成功和失败都保留，不能只展示成功截图。

## 展示

- 首屏显示样本量、覆盖城市/设备、最后测试日与局限。
- 筛选器：城市、设备、OS、SIM 形态、当地网络、日期区间。
- 状态采用“本次成功/本次失败/未测试/证据不足”，不用永久“支持/不支持”。
- 每条结果可展开方法和脱敏证据；聚合统计显示分母。

## 信任、Schema 与内链

- 公开方法版本、数据字典、修订和纠错入口。
- 数据集可用 `Dataset` 仅在真实可下载/查询且字段可见时；页面仍以 `CollectionPage` 为主。
- 入链：pillar、网络排障、漫游成本页；出链到 OTP 状态板解释平台层差异。

## 验收

- 无原始样本时返回 noindex 的方法预告，不伪装已验证。
- 聚合分子/分母可从原始记录重算。
- 投稿进入线索池，未经复核不计入结论。
- 明示小样本、地域/时间偏差和不保证未来结果。
