# Brief：验证码兼容性状态板

- 建议 URL：`/research/otp-compatibility-status/`
- 主关键词：`giffgaff 验证码兼容性状态`
- 意图：查看某平台在特定账号/设备/网络环境和时间窗口下的样本，不查询“永久可接码名单”。
- 发布门槛：有真实测试与方法；不能把匿名评论数量当成功率。

## 与网络矩阵的边界

网络矩阵回答普通运营商层是否工作；本页只在普通短信/官方短信已确认后，记录单个平台 OTP 请求结果。通用无信号和短信排查全部回 `/guides/4-signal/`。

## 数据字段

- 平台名称、公开政策/帮助链接与测试日期。
- 新注册/登录/恢复/高风险操作等请求类型。
- 号码输入格式、国家/城市级环境、设备/OS、网络类型。
- 账号是否新建、请求次数与等待窗口（不保存可识别账号信息）。
- 结果：收到、超时、平台明确拒绝、限频、未完成。
- 样本来源、证据等级、失败文案的脱敏摘要。

禁止发布验证码内容、绕过 KYC/反滥用步骤、账号买卖、IP 规避或批量接码建议。

## 展示与统计

- 显示“最近样本”而非永恒支持列表。
- 每个平台展示成功数/总样本数、日期范围、环境差异和证据等级。
- 样本不足时明确“证据不足”，不把 1/1 写成 100% 兼容。
- 平台政策变化可将旧记录标为 historical，不删除失败记录。

## 页面结构

1. 直接答案与限制。
2. 方法、样本量与最后更新。
3. 平台筛选和状态卡。
4. 普通短信 vs OTP 决策树。
5. 如何提交脱敏线索。
6. 数据字典、来源、纠错与修订。

## Schema、隐私与验收

- 有真实数据时可用 `Dataset`/`CollectionPage`；FAQ 必须与页面可见问答一致。
- 不接收号码、邮箱、用户名、密码、验证码、截图中的个人数据。
- 聚合可重算；机器人/重复投稿去重方法公开。
- 入链：pillar、网络排障、平台证据门禁页；平台页只有独特数据时才独立索引。
