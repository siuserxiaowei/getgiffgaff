# Brief：中国漫游成本与防误扣费工具

- 目标 URL：复用并重写 `/guides/5-travel-data/`，不新建同义页。
- 主关键词：`giffgaff 中国漫游成本`
- 意图：根据操作当日官方费率和用户预计用量试算，不在教程里写死价格。

## 信息增益与边界

本工具解决“我主动发一次短信、打一段电话或误用数据会花多少”的输入问题。它不预测未来费率、不承诺当地网络覆盖，也不把漫游动作等同于第三方 OTP 成功。

## 费率配置

每个费率字段必须包含：

- `serviceType`：outgoing SMS、outgoing call、incoming call、data 等。
- `rate`、`currency`、`billingUnit`、最小计费单位。
- `sourceUrl`、`checkedAt`、适用国家/区域与条件。
- `status`：verified、stale、unknown。

若任一关键费率为 stale/unknown，计算按钮改为“打开官方页面核对”，不能返回伪精确总价。

## 用户输入与输出

- 预计 SMS 条数、通话分钟、数据 MB。
- 用户自定义 GBP/CNY 汇率；默认汇率必须带来源和时间，或保持空白。
- 输出按服务分项、GBP 合计、可选 CNY 参考值、最大误差提示。
- 数据输入必须有高费用警示和“关闭数据漫游/后台数据”的防误扣费清单。

## 页面结构

1. 当前费率核验状态与直接答案。
2. 计算器。
3. 账单单位和舍入解释。
4. 中国境内注册网络与手动选网的官方入口。
5. 保号动作选择与 OTP 无关性。
6. 来源、方法、FAQ、修订记录。

## 技术、Schema 与测试

- 配置与页面同版本发布，费率过期门禁可自动阻止部署或隐藏结果。
- 使用客户端计算，不采集号码、城市精确位置或账号数据。
- 使用 `WebApplication`/`WebPage`；不发布无证据 `Offer`。
- 测试小数、最小计费单位、0 用量、异常大数据量、过期费率与汇率空值。

## 内链

- 入链：pillar、保号页、网络/OTP 排障页。
- 出链：官方 roaming、保号规则、网络排障；交易 CTA 不放在高费用警示旁。
