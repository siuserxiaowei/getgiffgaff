# Brief：G0/G2 总成本计算器

- 建议 URL：`/tools/g0-g2-total-cost/`
- 主关键词：`giffgaff G0 G2 总成本`
- 搜索意图：在付款前用自己的数值比较两条路径，而不是再读一篇主观推荐。
- 集群 owner：比较结论仍归 `/answers/`；本页只负责输入、公式、结果与敏感性分析。
- 状态：计划中的原创工具；没有可审核公式前不发布。

## 信息增益

竞品普遍把“卡价”当总成本，或用单一静态价格得出推荐。本工具必须让用户覆盖全部默认值，并分别显示现金成本、时间成本和不确定性，不能把 G2 写成新手或急用的默认选项。

## 必须输入

| 字段 | 类型 | 规则 |
|---|---|---|
| 卡与运费 | CNY 数值 | 用户输入；默认值只有在有当前销售证据时才提供 |
| 首次 Credit/plan | GBP 数值 | 来源和核验日紧邻字段；允许填 0，但解释原因 |
| 支付/兑换附加成本 | CNY 数值 | 不引导绕过支付校验 |
| 预计操作时间 | 小时 | G0/G2 分开输入 |
| 用户时间价值 | CNY/小时 | 默认空白，不替用户估值 |
| 未来 12 个月主动维护预算 | GBP 数值 | 不把某个动作写成永久保号承诺 |
| 风险缓冲 | CNY 数值 | 用于丢件、付款失败或恢复成本，由用户决定 |

## 公式与输出

公开显示公式：

`总成本 = CNY 现金项 + GBP 项 × 用户输入汇率 + 时间小时 × 时间价值 + 风险缓冲`

输出必须包括：

1. G0、G2 分项合计与差值。
2. 现金成本和时间成本分开显示。
3. 汇率、维护预算和时间价值变化时的敏感性提示。
4. “无法据此判断账号/条款合规或 OTP 送达”的边界。
5. G2 结果旁固定显示来源、本人登记、控制权和当前条款检查清单。

## 页面结构

1. 直接答案：计算器比较的是用户输入，不是实时报价。
2. G0/G2 定义与本站销售关系披露。
3. 计算表单。
4. 结果、公式与假设。
5. 条款/控制权停止条件。
6. 如何复核官方 Credit 与漫游信息。
7. FAQ 与修订记录。

## 证据与更新

- 价格字段必须保存 `sourceUrl`、`checkedAt`、币种和适用条件。
- 没有实时证据时默认值留空，不能用旧截图填充。
- 每次价格配置变化写修订记录；用户输入只在浏览器内计算。

## 隐私、安全与 Schema

- 不收集姓名、邮箱、手机号、账号、支付卡、密码或验证码。
- 禁止提供虚假地址、支付规避、共享账号或他人身份建议。
- 使用 `WebPage`/`SoftwareApplication` 仅描述页面可见功能；无实时销售证据时不输出 `Offer`、`Review`、`AggregateRating`。

## 内链与 CTA

- 入链：pillar、`/answers/`、`/shop/`、`/guides/1-order/`。
- 出链：结果解释回 `/answers/`；决定购买后才去 `/shop/`；条款不清楚时停止购买，不用 CTA 推进。

## 验收

- 公式有自动化测试，空值、负数、极大值和币种舍入均有边界。
- 所有默认值可追溯且带日期；无证据时界面明确“请自行输入”。
- JS 关闭时仍显示公式、字段定义和人工计算方法。
- canonical、自指 OG、200、indexable、无敏感查询参数缓存。
