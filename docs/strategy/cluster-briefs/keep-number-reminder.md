# Brief：giffgaff 保号提醒

- 建议 URL：`/tools/keep-number-reminder/`
- 主关键词：`giffgaff 保号提醒`
- 意图：把最近一次官方列明动作换算成缓冲提醒日和最晚复核日。
- 集群 owner：规则解释归 `/guides/3-usage/`；本页只做本地日期计算、记录模板与 `.ics` 导出。

## 核心原则

官方当前规则是至少每 6 个月完成列明的使用/购买动作；“第 5 个月提醒”只是本站风险缓冲，不是新的官方周期。单纯接收短信不作为默认有效动作。

## 输入与计算

- 最近一次可验证动作日期，必填。
- 动作类型：主动通话、主动 SMS/MMS、移动数据连接、购买 Credit/plan、未知。
- 缓冲月数/天数，默认 5 个月但可修改。
- 用户时区，仅用于日历展示。

输出：

1. 第 4 个月预提醒。
2. 第 5 个月操作提醒。
3. 按 6 个月规则计算的“必须重新核对官方页面”日期。
4. 当前动作是否属于官方页面列明类型；未知时不下结论。
5. 可下载的本地 `.ics`，标题不包含号码或账号。

## 页面结构

1. 直接答案与“收到短信不等于官方列明动作”。
2. 日期计算器。
3. 官方动作与不计入/未知动作说明。
4. 维护日志模板：日期、动作、可见结果、来源 URL、核验日。
5. 已停用时立即去官方支持的边界。
6. FAQ、来源和修订记录。

## 技术与隐私

- 全部计算在浏览器本地完成；不上传日期、号码或账号。
- localStorage 默认关闭；若用户主动保存，要有清空按钮和说明。
- `.ics` 只含通用标题、日期和本站/官方来源链接。
- 日期运算使用日历月而不是固定 180 天，并在界面说明差异。

## 证据与 Schema

- 直接链接当前 inactive/deactivation 帮助页、Credit 与漫游页。
- 官方规则旁显示 `checkedAt`，本站缓冲建议必须单独标识。
- 可用 `WebApplication` + 可见 FAQ；不使用 HowTo 冒充官方保证。

## 内链与验收

- 入链：pillar、`/guides/3-usage/`、`/qa/02-topup/`、充值服务页。
- 出链：保号规则、当前漫游页、官方支持与 Contact 的订单边界。
- 测试月末、闰年、时区、未来日期、未知动作和空输入。
- 不出现“永久保号”“20 年保号”或成本保证。
