# 第一波：只做 4 个页面 owner（其中 1 个为现有页）

## 结论

第一波不做几十篇，也不做“海外账号 KYC 大全”。建议上线/维护 **4 个 owner**：保留并扩写 1 个现有短信诊断支柱，新建 3 个 Claude 页面。ChatGPT 泛 KYC、海外账号 KYC、海外账号换手机号全部舍弃；ChatGPT 登录/手机号验证暂缓。

优先级依据：Bing Webmaster 3 个月观察中，`Claude KYC` 667 impressions，`Claude 封号` 922，相关封号变体更高；`ChatGPT KYC` 仅 17 impressions 且 SERP 意图混杂。此数据合并主线后应从 `docs/research/bing-webmaster-live-2026-07-20.md` 复核。

## 1. P0 维护：`/guides/4-signal/`

**角色**：短信/OTP 诊断支柱。

**主词**：`giffgaff 收不到验证码`。

**合并词**：`英国手机号收不到验证码`、`英国号码收不到短信`、`giffgaff ChatGPT 验证码`、`giffgaff Claude 验证码`。

现页已经正确区分：

- 无信号/所有短信失败：SIM、设备、账号、漫游、当地网络。
- 普通短信成功、单平台 OTP 失败：平台号段策略、账号、设备、请求频率或发送路由。

只需在下一轮实施中补：

1. H2 或 FAQ 明确包含“英国手机号收不到验证码”；不要另建泛英国号码页。
2. 平台分流新增 Claude 电话验证的官方要求摘要，但不做兼容承诺。
3. 链到 `/research/otp-status/` 解释“个案不等于永久兼容”。
4. CTA 只在用户确认需要长期可控英国实体号码后出现；不得写“买 G2 就能收 Claude/ChatGPT 验证码”。

## 2. P0 新建：`/guides/claude-identity-verification/`

**角色**：Claude 账号验证小支柱。

**主词**：`Claude 身份验证`。

**合并词**：`Claude KYC`、`Claude 实名认证`、`Claude 刷脸认证`、`Claude 身份验证失败`、`Claude KYC 失败`。

建议标题：**Claude 身份验证 / KYC：证件、自拍、失败处理与隐私边界**

内容边界：

1. 首屏定义：官方身份验证是政府签发的带照片实体证件 + 可能的实时自拍；不是短信手机号验证。
2. 为什么出现：只引用官方“部分用例 / 平台完整性 / 安全与合规措施”，不猜测个人触发原因。
3. 准备项、可接受/不可接受证件类型、失败的官方重试与支持入口。
4. Persona 与 Anthropic 的数据角色、保存/访问范围，只按官方原文概括。
5. “验证后账号禁用”只给摘要并链向申诉页。
6. 显眼安全框：本人真实资料；不提供假身份、借证、证件生成、绕过或代验证。

直接官方来源：

- [Claude 上的身份验证](https://support.claude.com/zh-CN/articles/14328960-claude-上的身份验证)，本次核验 `2026-07-20`，源页 `dateModified=2026-06-17T20:15:55Z`。

转化：不放首屏购买 CTA。末尾只有“如果你实际卡在短信验证码，去看手机号验证/普通短信诊断”；这是任务分流，不暗示 SIM 能解决证件 KYC。

## 3. P0 新建：`/guides/claude-phone-verification/`

**角色**：Claude 短信手机号验证 spoke，也是离购买最近的新页面。

**主词**：`Claude 手机号验证`。

**合并词**：`Claude 电话验证`、`Claude 验证码收不到`、`Claude 更换手机号`、`电话号码被使用太多次`。

建议标题：**Claude 手机号验证：收不到六位验证码、号码限制与换号规则**

必须覆盖：

1. 官方电话验证是短信发送六位验证码，用于确认能访问该号码。
2. 当前官方只接受支持地区号码，且不接受 VoIP、Google Voice、应用生成号码、座机或其他无法收短信的号码；无法跳过。
3. 未收到验证码、发送错误、收到但失效、号码被使用过多的官方 FAQ 分支。
4. 官方当前说明“验证后无法更改关联号码”，因此应使用本人长期控制的号码。
5. 链到 `/guides/4-signal/` 做普通短信基线：卡能收普通短信不等于 Claude 必发 OTP。
6. 不声称英国/giffgaff 一定属于用户当前可用的支持组合；发布前复核 Claude 支持地区。

直接官方来源：

- [验证您的电话号码](https://support.claude.com/zh-CN/articles/8287232-验证您的电话号码)，本次核验 `2026-07-20`，源页 `dateModified=2026-05-19T19:07:50Z`。

转化：正文完成官方资格判断与普通短信诊断后，才给两个弱 CTA：查看 `/shop/` 当前卡片说明；到 `/contact/` 咨询卡状态/普通短信排查。CTA 明示客服不能代过验证，用户不得发送密码、验证码、证件或 Cookie。

## 4. P0 新建：`/guides/claude-account-disabled-appeal/`

**角色**：高需求的合法恢复 spoke。

**主词**：`Claude 封号`。
**合并词**：`Claude 账号被封`、`Claude 被封 403`、`Claude Code 封号`、`Claude 账号申诉`。

建议标题：**Claude 账号被禁用 / 403：先判断状态，再走官方申诉**

内容顺序：

1. 答案优先：403 不自动等于封号。先记录官方提示，区分个人账号禁用、组织暂停、登录/客户端技术错误。
2. 只有官方明确禁用/终止时，才进入申诉路径；只链接官方提交申诉。
3. 说明被禁用用户的官方数据导出/删除边界，以及组织暂停的独立分支。
4. 不推测“封号潮数字”或把社区相关性写成确定原因。
5. 禁止“换手机号/换新卡/换账号/换地区即可恢复”、批量重开或规避政策内容。

直接官方来源：

- [保障措施警告和申诉](https://support.claude.com/zh-CN/articles/8241253-保障措施警告和申诉)，本次核验 `2026-07-20`，源页 `dateModified=2026-07-09T14:33:08Z`。
- [Claude 上的身份验证](https://support.claude.com/zh-CN/articles/14328960-claude-上的身份验证)，用于“验证后禁用”与官方申诉衔接。

转化：**不放购买/微信硬 CTA**。新号码不是解封工具。页尾可以回到 Claude 身份验证页或官方支持；保持品牌可信度比强行卖卡更重要。

## 明确不做

| 候选 | 决定 | 原因 |
|---|---|---|
| `/guides/chatgpt-kyc/` | 不建 | 17 impressions/3M；SERP 同时是账号实名、网络安全可信访问、KYC GPT、金融自动化；与登录失败/手机号验证均 0/10 重合 |
| ChatGPT 身份验证失败 | 暂缓 | 实际是登录认证故障；与手机号验证/KYC 不同，当前没有足够趋势支撑本轮 |
| ChatGPT 手机号验证 | 暂缓 | SERP 被接码/“100%成功”占据，合规风险高；无足够 Bing 趋势 |
| `/guides/overseas-account-kyc/` | 不建 | SERP 主要为银行、支付、企业开户等 YMYL；跨平台规则不可通用 |
| `/guides/overseas-account-change-phone/` | 不建 | SERP 按华为、网易、Apple ID、ChatGPT 等平台分裂；通用页没有稳定 owner |
| 每个平台各写一篇 giffgaff 验证码 | 不建 | 本站 `/research/otp-status/` 尚未达到样本门槛；会制造“某平台可用”的不实暗示和大量薄页 |

## 发布顺序与门槛

1. 先更新 `/guides/4-signal/` 的近义词覆盖与分流。
2. 发布 Claude 身份验证 hub，同时发布 Claude 手机号验证，确保概念可双向解释。
3. 发布 Claude 封号/申诉页并从身份验证页链接过去。
4. 每页发布前重新访问直接官方来源，记录核验日；规则冲突时先 noindex/暂停发布。
5. 30 天后只用自有 Bing/GSC 查询和转化事件决定是否继续，不按词表批量扩写。

成功不是“发了多少篇”，而是：查询落到唯一 owner、ChatGPT/Claude/KYC/手机号验证不互相蚕食、没有违规绕过内容，以及手机号页能产生合格的选卡/咨询点击而不承诺 OTP。
