# 站外文章平台发布操作手册

更新日期：2026-07-20

本手册只覆盖当前可条件发布的三篇教育型内容：06 英国手机卡六问、01 保号第 5 个月检查法、02 验证码二分诊断。它不授权支付、买链、群发外联、绕过登录/扫码/验证码或发布被阻塞的内容。

发布优先级、准确 URL 和所有禁止项见 [README](README.md)。如果没有时间记录发布 URL 或数据，直接不记录即可；空白台账不是必需动作，也不会提高排名。

## 先做共同的 8 步检查

每次只准备一篇、一个平台版本。完成这 8 步后才进入对应平台的发布步骤：

1. 确认选的是 06、01 或 02；03、04、05 不进入本轮发布队列。
2. 打开该文章包，按其“推荐链接与发布版本”取得唯一深层页 URL；不要改成主页、商品页、二维码、Contact 页或多个链接。
3. 逐一打开文中官方来源并在当天复核。来源打不开、内容变化、费率/流程不确定时，停止该稿，不用旧截图替代。
4. 用生产域打开深层页，确认它仍能解释文章承诺的那个问题；不把页面存在误写成搜索已收录或会带来咨询。
5. 准备当天的公开来源截图和原创图卡。截图只保留必要公开文字、URL 和核验日期；不展示真实账号、号码、订单、余额、验证码、支付资料、SIM 序列号或 eSIM 凭证。
6. 在文章第一次提到 getgiffgaff、深层页或咨询语境前，放入以下身份披露：

   > getgiffgaff 是独立第三方中文教程与销售服务站，不是 giffgaff Limited 官方网站、官方客服或授权代表。本文提供的是教程或选择方法；运营商规则以发布当天重新核验的官方页面为准。

   若内容出现销售、咨询或商业入口，再紧接着放入：

   > 商业披露：本文涉及 getgiffgaff 自有服务/销售入口；正文中的运营商事实均链接至 giffgaff 官方来源。

7. 删除任何“保证有信号”“OTP 必到”“永久保号”“官方合作”“最低价”“真实库存”“亲测成功率”等无证据说法。
8. 在手机预览中检查标题、图片、披露、官方来源与唯一深层页链接；确认没有把读者导向不安全的支付、二维码或私聊关键词引导。

## 公众号：条件发布操作

建议首篇：06 英国手机卡六问；之后才是 02 或经过重写的 01。

1. 在公众号官方后台用账号所有者的设备登录/扫码。若当天无法登录，停止发布，不用第三方代发或非官方工具绕过。
2. 选择一个已通过共同检查的文章包。公众号正文可使用完整长文版，但不能复制小红书的首屏、图片顺序和结尾。
3. 配置一张原创封面，以及 2–4 张当天官方来源截图或自制流程图。每张图有能读懂的标题或图注；不伪造运营商后台或真实订单。
4. 在首段或第一次提及本站之前放身份披露。若本文出现咨询/销售语境，在同一可见区域放商业披露。
5. 在读者已经理解判断方法后插入唯一深层链接：

   | 文章 | 公众号 URL |
   | --- | --- |
   | 06 英国手机卡六问 | `https://getgiffgaff.com/guides/8-uk-sim-choice/?utm_source=dist_wechat_official` |
   | 01 保号第 5 个月检查法 | `https://getgiffgaff.com/tools/keep-number-reminder/?utm_source=dist_wechat_official` |
   | 02 验证码二分诊断 | `https://getgiffgaff.com/guides/4-signal/?utm_source=dist_wechat_official` |

6. 不再插入商城、SKU、收款码、客服二维码、私信关键词或第二个站内链接。读者进入深层页后，可自行决定是否使用站内咨询入口。
7. 用手机预览检查链接、长图裁切、披露是否在折叠前可见、官方链接是否可打开；确认无误后再发布。
8. 发布后只在确有问题时更新事实或更正链接。可选地保存真实文章 URL 和发布时间；这只是复盘信息，不是 SEO 动作。

## 小红书：条件发布操作

建议首篇：01 保号第 5 个月检查法；06 只能在做出实质不同的图卡、首屏和例子后作为第二平台版本发布。

1. 在小红书官方 App 或后台用账号所有者的设备登录/扫码。无法登录时停止，不使用多个账号、自动化或绕过审核。
2. 先确认当天这个账号和内容形态是否支持可点击外链。不能放链接时，只写准确的工具或站点名称，例如“getgiffgaff 保号提醒工具”；不要在评论区反复补链接，也不要用“私信关键词”批量引导。
3. 为每篇内容制作 6–9 张原创图卡：问题、官方边界、操作清单、深层页的中立用途、限制条件。首图不使用“必到”“永久”“零成本”“官方”“最好”等承诺。
4. 使用与公众号不同的标题、首屏、结构、例子、配图和结尾。不能只替换几个同义词后重复发布全文。
5. 在正文首次提到本站、工具或链接前加入身份披露；出现咨询/销售语境时加入商业披露。不要把披露只放在最后一张图或评论区。
6. 仅在平台允许时放一个对应链接：

   | 文章 | 小红书 URL |
   | --- | --- |
   | 06 英国手机卡六问 | `https://getgiffgaff.com/guides/8-uk-sim-choice/?utm_source=dist_xiaohongshu` |
   | 01 保号第 5 个月检查法 | `https://getgiffgaff.com/tools/keep-number-reminder/?utm_source=dist_xiaohongshu` |
   | 02 验证码二分诊断 | `https://getgiffgaff.com/guides/4-signal/?utm_source=dist_xiaohongshu` |

7. 预览时逐张核对中文是否清晰、图注与正文是否一致、没有真实敏感信息，也没有购买、价格、库存、二维码或 OTP 保证。
8. 发布后正常回答与文章直接相关的真实提问；不复制粘贴链接回复、不追着用户私信、不把评论区当外链发布区。

## 知乎：当前阻塞，不发布

现有 [知乎回答草稿](../outreach/drafts/zhihu-answer.md) 是 `BLOCKED`。当前不能把 06、01 或 02 改成知乎回答来“试一试”，也不能仅因能登录就发布。

未来如需解锁，顺序必须是：

1. 由账号所有者确认账号类型。
2. 用该账号在官方渠道读取当日全部适用规则，特别核对商业内容、外链、披露和重复分发限制。
3. 留存核验日期、规则 URL 和适用范围，由独立复核人确认草稿、披露和链接方式符合规则。
4. 重新跑共同 8 步检查，并确认页面 release、身份和事实门禁。
5. 以上全部通过后，才可按“前 150 字先回答问题与边界 → 判断标准 → 操作顺序 → 常见误区 → 一个干净 URL”的结构重新制作知乎专用稿；知乎只使用无 UTM 的 canonical URL。

在第五步之前，正确动作是保持不发布，不是换账号、拆成多篇近义回答，或在别人回答下追加推广链接。

## 今天开始的 7 天节奏

这是低频、可停止的节奏，而不是每天刷内容。某天的发布条件未通过，就把那天变成准备/复核日，不补发、不赶进度。

| 天数 | 当天要做的事 | 允许发布的条件 | 明确不做 |
| --- | --- | --- | --- |
| 第 1 天（今天） | 登录/扫码公众号和小红书；复核 06 的官方来源、制作公众号封面与来源图 | 所有共同 8 步通过后，发布 06 公众号版 | 不发知乎；不发 03/04/05；不放多个站内链接 |
| 第 2 天 | 手机打开已发文章和深层页，核对链接/披露；准备 01 的 6–9 张原创图卡 | 不要求新发；只在事实错误时更正 | 不在评论区反复贴链接；不把访问数当排名 |
| 第 3 天 | 复核 01 的官方保号规则和工具边界，完成小红书版重写 | 共同检查通过且账号允许时，发布 01 小红书版 | 不写“自动保号”“永久保号”；不索要用户真实日期/账号资料 |
| 第 4 天 | 查看真实留言和聚合来源数据，准备 02 的公众号版本与当天截图 | 不要求新发 | 不从一两天数据判断流量恢复；不向外站群发外联 |
| 第 5 天 | 复核 02 的网络/手动选网/短信政策，手机预览文章 | 共同检查通过后，发布 02 公众号版或已实质重写的小红书版 | 不承诺验证码送达；不提供绕过风控、批量接码或平台兼容名单 |
| 第 6 天 | 若 06 的小红书图卡、案例和结尾都与公众号版实质不同，则制作第二平台版本；否则只补素材 | 仅在重写、图卡、披露和平台链接能力都通过时，发布 06 小红书版 | 不复制同一全文；不为了频率强行发布 |
| 第 7 天 | 汇总真实已发布 URL、内容修改、聚合页面访问/联系点击和实际收到的咨询数；决定下周是否继续 | 不要求新发。只有连续完整数据足以支持时再调整选题 | 不把台账、点击或短期波动当作排名或成交结论；不解锁 03/04/05/知乎 |

如果公众号或小红书当天仍无法登录，顺序不变：只完成复核、图片与正文准备，等账号所有者能安全登录后再发布。不要尝试代替扫码、借号、购买账号或使用自动发布工具。

## 编辑采用与真实外链的正确做法

站外内容发布后的下一步不是群发 pitch。独立编辑是否引用深层页，必须由其自身读者需求和明确政策决定。

可以准备的三个中立资源是：

| 资源 | 合适的读者任务 | 可由编辑自主采用的自然描述 |
| --- | --- | --- |
| `/guides/8-uk-sim-choice/` | 留学、旅行或英国生活前的选择条件核对 | “英国手机卡六问选择框架” |
| `/guides/7-arrival-checklist/` | 收到 UK SIM 后按阶段验收 | “UK SIM 到手七步验收清单” |
| `/tools/keep-number-reminder/` | 低频使用者建立第 5 个月复核提醒 | “浏览器本地保号提醒工具” |

只有在目标站明确允许、事实和资产门禁完成、独立第三方销售身份已披露时，才可逐一发送一封定制的资源建议。编辑可选择不链接、链接官方来源、使用任意锚文本或拒绝；不得购买、交换、要求 dofollow、指定关键词锚文本、批量发送或在评论/GitHub Issue 中推广。
