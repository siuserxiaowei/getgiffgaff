# 证据缺口与待复核项

## G-01 — OpenAI 独立身份/年龄核验流程

- 缺失问题：截至 2026-07-20，哪些 ChatGPT 用户、国家或能力会触发政府证件、自拍或第三方年龄核验？
- 为什么重要：中文“ChatGPT KYC”容易把年龄政策、MFA、登录异常和真实身份核验混为一谈。
- 已尝试：OpenAI 官方文档 MCP、OpenAI Help Center 已知 URL 与站内页面回读。
- 阻塞：没有定位到稳定、当前且可审计的专页正文；旧文章 ID 会复用/改题，不能靠记忆或旧 slug 下结论。
- 最小下一步：发布相关页前在 OpenAI Help Center 官方搜索中重新定位当日专页并保存页面标题、canonical URL 与短摘录；找不到就继续明确写“官方公开证据不足”。

## G-02 — 第三方平台对 giffgaff 号段的实时接受率

- 缺失问题：各平台对具体 giffgaff 号码/号段、漫游状态和中国境内 OTP 的当前成功率。
- 为什么重要：这是用户最想知道、也最容易被营销夸大的部分。
- 已尝试：平台官方帮助与 giffgaff 官方通信帮助。
- 阻塞：平台不公布号段级风控规则；个案成功/失败受账号、设备、IP、请求频率、号码历史和短信路由影响，且用户样本涉及私有数据。
- 最小下一步：只记录经过用户明确同意、去标识化的支持工单结果，按平台/日期/普通短信是否正常/错误文案做小样本观察；不得据此承诺通过率。

## G-03 — 中国漫游接收 SMS 的明确资费句子

- 缺失问题：giffgaff 当前官方资费页是否有可直接引用的“中国接收 SMS 收费为 £0”文字。
- 为什么重要：用户常问“收验证码是否收费”。
- 已尝试：rest-of-world 帮助页和 roaming charges 入口。
- 阻塞：本轮读取的帮助页主要说明套餐/credit、主动业务和数据，没有抓到足够稳定的中国行级接收 SMS 价目摘录。
- 最小下一步：发布资费文案前人工打开 https://www.giffgaff.com/roaming-charges ，选择 China 并留存当日可审计行；在此之前不写具体 £0 或绝对免费。

## G-04 — Claude 电话验证与身份核验的触发变化

- 缺失问题：Claude 是否会按地区、账号批次或产品表面改变注册手机号要求，以及哪些具体能力触发身份核验。
- 为什么重要：两篇官方页均是动态政策，且身份页明确说在逐步推出。
- 已尝试：Claude 官方 phone、identity、age、login 与 supported locations 页面。
- 阻塞：官方不披露完整风控条件；不能由帮助页推断每个用户必然遇到相同流程。
- 最小下一步：页面上线前与每月例行复核 ANT-01/02/03；所有正文用“可能触发/当前官方称”，不用“所有账号”。

## G-05 — Claude 封禁的逐案原因与恢复结果

- 缺失问题：某个用户的封号原因、申诉成功率和处理时长。
- 为什么重要：Bing 显示“Claude 封号”需求强，但这是高风险求助意图。
- 已尝试：Claude 身份验证官方页中的可能原因与申诉入口、登录帮助页。
- 阻塞：官方只提供可能原因和申诉渠道；个案原因属于平台/用户私有数据，没有公开成功率。
- 最小下一步：内容只提供官方申诉入口与真实资料准备清单；不承诺解封、不推测原因、不销售“解封服务”。
