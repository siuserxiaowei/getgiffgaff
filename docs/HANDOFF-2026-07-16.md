## 接手后的准确执行顺序

1. 在 Cloudflare 第一条重定向规则中追加上述 5 个路径。
2. 执行 `npm run postdeploy`，保存完整输出。
3. 补一次生产 smoke：GET/HEAD、双斜杠、`index.html`、普通 query、3 个 noindex 增长页。
4. 门禁全绿后 fast-forward 合并并推送 `main`。
5. 必要时执行 `npm run submit:indexnow` 重新通知 39 URL。
6. 登录 GSC、Bing、百度后台确认 sitemap 回读；百度遇到实名/OTP 时交由账号所有者完成。
7. 业务方提供快团团直达链接后替换二维码兜底，并用真机完成 G0/G2 测试单。
8. 生产门禁首次全绿后的下一个完整自然日作为 28 天基线 D1。
