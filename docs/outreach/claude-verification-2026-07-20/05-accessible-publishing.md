## 正文

当你准备把这批稿件粘贴进知乎、公众号、开发者社区或小红书后台时，应先从这份说明开始复核。它不授权自动登录、扫码、绕过验证码、绕过审核或批量分发。四份稿件均为草稿；目标平台规则、官方来源、最终落地页和披露任一未通过，就不发布。

## 一、统一内容边界

四份稿只能说明：

- Claude 官方当前把手机号验证与身份验证作为不同流程；
- 官方电话验证页对支持地区、号码类型、收不到码和换号有当前说明；
- 官方身份验证页对本人有效证件、自拍、失败与数据处理有当前说明；
- 账号被明确禁用时，用户可以按 Claude 官方页面提交申诉；
- 403 本身不足以证明账号已被封。

四份稿不得增加：

- “这个号段能过”“giffgaff 必收 Claude 验证码”“购买后包验证”；
- 临时/共享接码、VoIP 替代、批量账号、借证、假身份、证件生成、代拍；
- 切换地区、伪造位置、换号重开、规避封禁或绕过 Usage Policy；
- 未提供的亲测、成功率、用户案例、合作、排名、流量或咨询数据。

## 三、单一回链和 UTM 方案

生产站使用单一 `utm_source`。不要添加 `utm_medium`、`utm_campaign`、`utm_content`、`utm_term`，也不要写入文章名、作者名、群名、手机号或账号 ID。

| 实际渠道 | 允许形式 | 示例 |
|---|---|---|
| 微信公众号 | `dist_wechat_official` | `https://getgiffgaff.com/guides/claude-phone-verification/?utm_source=dist_wechat_official` |
| 小红书 | `dist_xiaohongshu` | `https://getgiffgaff.com/guides/claude-phone-verification/?utm_source=dist_xiaohongshu` |
| 微信社群 | `dist_wechat_group` | `https://getgiffgaff.com/guides/claude-phone-verification/?utm_source=dist_wechat_group` |
| 私聊分享 | `dist_private_share` | `https://getgiffgaff.com/guides/claude-phone-verification/?utm_source=dist_private_share` |
| 人工确认的合作发布 | `dist_partner` | `https://getgiffgaff.com/guides/claude-account-disabled-appeal/?utm_source=dist_partner` |
| 其他平台 / 未确认渠道 | 无参数 canonical | `https://getgiffgaff.com/guides/claude-identity-verification/` |

规则：

1. 每篇只放一个对应的最终 `/guides/` 落地页；官方来源链接不算本站回链。
2. 平台不支持外链时，保留准确页面名称，不在评论区补链、不诱导私信关键词。
3. 不要求发布者使用关键词锚文本、dofollow、固定位置或保留期限。
4. 有对价时，正文显眼商业披露；链接属性按发布者政策使用 `sponsored`，必要时同时 `nofollow`。
5. URL 目前是最终路径占位。发布前必须确认页面已上线、返回 200、canonical 正确且内容通过 release gate。

## 四、文字无障碍检查

- 标题先写问题，不只写“KYC”“403”等缩写；首次出现 KYC 时解释为身份验证语境。
- 不用颜色、表情或图片单独表达“允许 / 禁止 / 失败”；正文给出等价文字。
- 段落保持短，列表使用真实编号或项目符号；不使用全角空格模拟对齐。
- 链接文本说明目标，例如“Claude 官方《验证您的电话号码》”，不用“点这里”。
- 表格在移动端无法阅读时，改成逐项列表；不要用截图代替整段正文。
- 中英混排保留必要产品名，错误码写成 `403`，短信一次性验证码首次写作“短信验证码（OTP）”。
- 重要警告用完整句子：`不接码、不借证、不承诺通过。` 不只放图标。

## 六、披露与隐私

在首次出现本站名称、本站链接或销售/咨询语境之前或同屏位置放置：

> getgiffgaff 是独立第三方中文教程与销售服务站，不是 Anthropic、Claude 或 giffgaff 的官方网站、官方客服或授权代表。本文包含本站自有教程页链接；平台规则以发布当天重新核验的官方页面为准。

如果涉及卡片咨询或销售入口，再增加：

> 商业披露：本文涉及 getgiffgaff 自有服务/销售入口。本站不能代过身份或手机号验证，不保证第三方短信验证码送达；请勿发送密码、验证码、证件、自拍、Cookie 或完整账号截图。

运营者不得请求读者把账号秘密发到评论、私信、微信或工单。收到敏感信息时，不复制进仓库或运营台账，并按适用隐私流程处理。

## 七、逐步发布流程

1. 打开三类直接官方来源：身份验证、电话验证、保障措施申诉；开发者社区稿另开 Claude Code 排障页。
2. 对照稿件逐条复核。任何来源打不开或规则冲突，删除/更新相关段落；无法确认则停止发布。
3. 打开单一落地页，确认 HTTP 200、canonical、移动端正文、官方来源、身份披露和咨询边界。
4. 根据实际渠道选择单一 URL；不要预填另一个渠道的 `utm_source`。
5. 上传原创封面/图卡和当日官方截图；逐张填写替代文本或在正文紧邻位置给出等价描述。
6. 把披露放到首次本站链接前；用手机预览字号、对比度、表格、链接和折叠位置。
7. 人工确认没有敏感信息、绕过建议、通过承诺或多余本站链接后再提交审核。
8. 只记录真实发布日期和公开文章 URL。不要把点击、点赞、咨询或收录预填为成功。
