## Go / No-Go 屏幕卡文案

### Go 候选：五项必须同时成立

- 最新版 giffgaff 官方 App 可用。
- 精确机型和地区版本有兼容依据。
- 设备确认无运营商锁。
- 账号和邮箱由本人控制，恢复路径可用。
- 明确接受新 eSIM 生效后旧 SIM 停止工作。

Go 只表示可以按官方 App 路径继续，不是转换成功、网络可用或 OTP 到达保证。

### No-Go：任一项成立就先停

- 只能依赖第三方 QR、SM-DP+、写卡或修改版 APK。
- 设备精确版本兼容性不明，或设备仍有网络锁。
- 无法使用最新版官方 App。
- 账号、邮箱或恢复路径不受本人控制。
- 不能接受旧 SIM 失效，或正在等待重要短信/OTP。
- 只有单一设备/单一恢复通道，失败后没有安全退路。

低风险下一步：继续使用实体 SIM、先恢复账号、处理设备锁、核验精确型号，或更换明确兼容设备。

## 必须保留的负面测试

成片和文字稿必须保留以下结果，不得只展示成功路线：

1. 精确型号或地区版本缺证据 → No-Go / 需核对。
2. 设备显示运营商锁或锁定状态不明 → No-Go。
3. 最新官方 App 无法安装、登录或显示官方切换入口 → No-Go，不提供 APK 替代。
4. 账号、邮箱或恢复方式不可用 → No-Go。
5. 用户不接受旧 SIM 停止工作 → No-Go。
6. 普通短信成功但平台 OTP 失败 → 记录为平台层结果，不能归因为 eSIM 必然失败；同样不能用普通短信成功承诺 OTP。
