## 七步可打印清单

打印日期：`____________`　验收人：`____________`　内部记录 ID：`____________`

### 1. 核对订单与实物

- [ ] 收货日期和包裹外观已记录。
- [ ] 卡片数量、卡板和订单描述一致。
- [ ] 对外或共享记录只保留订单号后四位，不展示完整订单号、姓名、地址或电话。
- [ ] 如果数量、外观或订单描述不一致，先暂停后续操作。

结果：`[ ] 通过　[ ] 暂停　[ ] 需订单方处理`

### 2. 留下脱敏证据

- [ ] 拍摄包裹、卡板和异常位置的整体照片。
- [ ] 完整 ICCID、激活码、13 位 SSN、二维码、地址和其他可识别信息均已遮挡。
- [ ] 错误页面只保留错误文案、时间和必要界面，不包含密码、验证码或完整支付资料。
- [ ] 原始敏感照片未上传到公开页面、公共云盘或编辑共享稿。

结果：`[ ] 通过　[ ] 暂停　[ ] 需补拍脱敏证据`

### 3. 判断卡状态，避免误操作

- [ ] 已按订单描述确认这张卡预期处于未激活、已激活或其他明确状态。
- [ ] 若准备激活，已确认 giffgaff 官方路径可使用激活码或 13 位 SSN。
- [ ] 未在已有号码的账号中盲目激活另一张 SIM；官方说明提示，这可能启动 SIM swap。
- [ ] 状态与订单不一致时，没有连续重复激活或尝试多个账号。

结果：`[ ] 通过　[ ] 暂停　[ ] 需运营商或订单方判断`

### 4. 确认后续控制方式

- [ ] 已写清由谁登录账号、接收账号邮件、更新资料和执行恢复操作。
- [ ] 实际控制方式与订单或交付说明一致。
- [ ] 没有通过共享表格、聊天截图或邮件传播密码、验证码或完整恢复信息。
- [ ] 若控制方式模糊或不一致，先暂停充值、改资料和第三方平台绑定。

结果：`[ ] 通过　[ ] 暂停　[ ] 需书面确认责任边界`

### 5. 分开核对 Credit 与 plan

- [ ] 已在运营商 Dashboard 或 App 中读取当前显示，而不是只依据卖家截图或口头描述。
- [ ] 已分别记录 credit 和 plan 内含用量；官方说明将二者分开处理。
- [ ] 记录包含读取日期、时间和币种，但公开证据不包含完整账号信息。
- [ ] 未把 credit 余额直接等同于 plan 内含用量、人民币价值或未来可用金额。

结果：`[ ] 通过　[ ] 暂停　[ ] 需核对运营商显示`

### 6. 记录网络注册结果

- [ ] 使用一台已知能正常读取 SIM 的测试设备，并记录设备、系统、国家/城市和测试时间。
- [ ] 分别记录“已注册网络”“未注册网络”或“结果不确定”，不只写“有信号/没信号”。
- [ ] 没有把单次测试扩大为某城市、校园、路线或室内场景的覆盖保证。
- [ ] 已知 giffgaff 使用 O2 网络，且官方条款将覆盖描述为预测而非保证。

结果：`[ ] 通过　[ ] 暂停　[ ] 需继续排查`

### 7. 单独测试普通短信

- [ ] 在个人、非商业的正常使用场景中完成一次普通短信测试，并记录发送/接收方向、时间和结果。
- [ ] 普通短信结果与网络注册结果分开记录。
- [ ] 没有把普通短信成功写成银行、平台或其他第三方 OTP 必达。
- [ ] 没有进行自动、批量、商业或异常设备模式的短信测试。

结果：`[ ] 通过　[ ] 暂停　[ ] 需运营商或第三方平台排查`

### 验收结论

- [ ] 七步均有记录，订单与实际交付一致，可以进入下一项自主管理操作。
- [ ] 至少一项不一致，已暂停充值、资料修改或平台绑定，并保存脱敏证据。
- [ ] 已把问题分到正确责任方：订单内容由订单方处理；账号、号码状态、网络或计费规则由运营商解释；第三方 OTP 由对应平台与运营商共同排查。

本清单只提供检查顺序，不保证 SIM、网络、短信或第三方服务可用。
