## 正文开始

如果 Claude 注册页面要求输入六位短信验证码，最常见的无效操作是连续点击“重新发送”。

它把几个不同的问题混到了一起：SIM 能不能收普通短信、号码是否符合 Claude 当前要求、平台有没有接受本次请求，以及验证码是否已经过期。

更稳妥的排查只有两步：**先建立普通短信基线，再检查平台 OTP。**

> 这是一份故障定位清单，不是“可过验证号段”名单。任何实体号码都不能保证 Claude 接受，也不能保证验证码送达。

## 先看页面到底要求什么

Claude 的手机号验证和身份验证不是同一件事。

- 输入电话号码和六位短信码，是手机号验证；
- 拍摄政府签发的带照片证件，并可能拍摄实时自拍，是身份验证；
- 收到账号暂停或终止通知，则进入账号申诉。

手机号不能替代身份证件，上传证件也不能修复一条没有送达的短信。先确认页面类型，才能避免做错方向。

Claude 官方[电话验证说明](https://support.claude.com/zh-CN/articles/8287232-%E9%AA%8C%E8%AF%81%E6%82%A8%E7%9A%84%E7%94%B5%E8%AF%9D%E5%8F%B7%E7%A0%81)当前写明，验证通过短信发送六位代码，并对支持地区和号码类型设有限制。官方列出的不接受类型包括 VoIP、Google Voice、通过应用创建的号码、座机和其他不能接收短信的号码。

这是按发布日可见的官方页面整理的规则边界，不能代替平台对具体账号和号码的实时判断。

## 分支 A：普通短信也收不到

先不要继续请求 Claude 验证码。此时基础短信链路还没有建立，优先检查：

1. SIM 是否已经激活，号码是否仍在本人账号中；
2. 手机是否无网络锁，SIM 接触和卡槽是否正常；
3. 人在号码本国以外时，漫游是否开启并成功注册当地网络；
4. 手机是否能接收另一部普通手机发送的测试短信；
5. 换一台兼容、无锁设备后，症状是否相同；
6. 账号或号码是否有停用、余额或网络服务提示。

一次只改变一个条件，并记录测试时间。否则同时换手机、切网络、重发 OTP 后，即使突然收到短信，也无法知道是哪一步生效。

## 分支 B：普通短信能到，只有 Claude OTP 不到

这只能证明基础短信不是完全离线，不能证明 Claude 已经接受号码并成功发出了这次 OTP。

接下来检查：

- 号码国家是否在 Claude 当前支持范围；
- 号码类型是否符合官方要求，而不是 VoIP、应用号或座机；
- 国际格式和国家代码是否正确；
- 页面是否提示号码已使用、发送失败或请求受限；
- 是否在短时间内多次请求；
- 输入的是否为最新收到、仍在有效期内的验证码。

如果页面提示等待，应按当前官方说明等待后再试，而不是开多个窗口同时请求。频繁重发可能产生多个代码，既增加过期码混淆，也可能触发平台限制。

## 分支 C：代码收到了，但页面仍不接受

先确认三件事：

1. 输入的是最后一次请求对应的最新代码；
2. 没有把短信中的其他数字、空格或标点一起复制；
3. 页面没有显示代码过期、号码已使用或账号状态错误。

不要把验证码发到群里请别人“代输”，也不要把完整浏览器截图、Cookie 或账号密码交给卖卡方。验证码的作用就是证明你控制该号码，泄露它会破坏这个安全边界。

## 为什么“今天能收到一次”仍不等于适合长期使用

Claude 官方当前说明，完成电话验证后，关联号码不能直接更改。因此选择号码时，不应只看今天能否收到一条短信，还要考虑：

- 号码是否由本人长期控制；
- 是否能持续登录运营商账户；
- 是否了解号码的活跃和停用规则；
- 是否保留其他官方允许的账号恢复方式。

临时接码、共享号码或借用他人号码，即使偶尔出现一条短信，也不能证明未来仍由你控制。

对于需要长期保留的实体 SIM，“只收短信”也未必等于完成了运营商要求的保号动作。运营商活跃规则和 Claude 的验证规则是两套系统，必须分别核对。

## 一张可以照着做的排查表

| 现象 | 优先排查 | 不应先做 |
|---|---|---|
| 普通短信也不到 | SIM、设备、漫游、网络和号码状态 | 连续请求平台 OTP |
| 普通短信能到，Claude OTP 不到 | 支持地区、号码类型、格式、账号状态和频率 | 宣称卡坏或购买“包过号” |
| 收到代码但不接受 | 最新代码、有效期、页面错误 | 把验证码交给第三方代输 |
| 明确账号被禁用 | 保存通知并走官方申诉 | 换号重开或建立多个账号 |

## 适合与不适合

适合：页面明确要求六位短信码、普通短信或 Claude OTP 没有送达，需要按症状逐层排查的人。

不适合：寻找临时接码、共享号码、借证、伪装地区、批量注册或“验证码必到”承诺的人。
