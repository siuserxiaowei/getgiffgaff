## 正文开始

在终端里看到 Claude Code 返回 `403`，很容易立刻得出一个结论：账号被封了。

但 `403` 是访问被拒绝的技术线索，不是完整的账号状态说明。个人账号被明确禁用、组织被暂停、登录状态失效，甚至 Claude Code 的安装或身份验证异常，都可能表现为访问失败。

真正有用的第一步，不是换号码或重新注册，而是先回答：**拒绝发生在哪一层？**

> 本文只整理官方排障和申诉路径，不提供换号重开、地区伪装、共享账号或绕过平台规则的方法。

## 第一层：是否收到个人账号暂停或终止通知

如果 Claude 明确通知个人账号被暂停或终止，这才进入账号申诉分支。

此时应保留：

- 通知的原始文字和发生时间；
- 使用入口，例如网页、桌面端、Claude Code 或 API；
- 出错前最后一个能够正常完成的步骤；
- 已经按照官方文档做过的排查及结果。

不要把自己的猜测写成事实。例如，只看到 `403` 时，不要在申诉里写“系统已经确认误封”；没有收到政策原因时，也不要自行编造一个原因。

Claude 官方的[保障措施警告和申诉说明](https://support.claude.com/zh-CN/articles/8241253-%E4%BF%9D%E9%9A%9C%E6%8E%AA%E6%96%BD%E8%AD%A6%E5%91%8A%E5%92%8C%E7%94%B3%E8%AF%89)提供当前申诉入口。发布后如果入口发生变化，应始终从官方帮助页重新进入，不要依赖第三方保存的旧表单。

## 第二层：是个人账号，还是所属组织被暂停

个人账号和组织不是同一个状态对象。

官方帮助页说明，登录后的受限账户页面可能列出被暂停的组织。一个组织无法访问，不足以证明个人账号已经永久终止；反过来，能进入个人账户页面，也不能证明每个组织都处于正常状态。

记录问题时应把两者分开：

```text
个人账号：能否登录 / 是否有暂停或终止通知
组织：组织名称 / 是否被列为暂停 / 其他组织是否可用
入口：网页 / Claude Code / API
时间：日期、时间、时区
```

这个区分能减少支持人员来回追问，也避免把组织权限问题误写成“账号封禁”。

## 第三层：只有 403，没有任何禁用通知

这时先把它当作技术诊断问题。

按下面顺序记录：

1. Claude 网页能否正常登录；
2. 网页、桌面端、Claude Code 和 API 中，哪些入口失败；
3. 错误发生在登录、OAuth、令牌交换、请求发送还是某个项目内；
4. 完整错误文字和状态码，而不是只截下 `403`；
5. 同一账号是否能看到组织状态。

Claude 官方另有[Claude Code 安装和身份验证排障](https://support.claude.com/zh-CN/articles/14552646-%E6%8E%92%E6%9F%A5-claude-code-%E5%AE%89%E8%A3%85%E5%92%8C%E8%BA%AB%E4%BB%BD%E9%AA%8C%E8%AF%81%E9%97%AE%E9%A2%98)。没有禁用通知时，应先完成与当前症状相符的官方步骤，再决定是否联系支持。

不要从来源不明的“防封脚本”开始。它既不能证明根因，也可能引入新的凭证和供应链风险。

## 截图和日志：够诊断，但不泄露秘密

支持材料需要能复现问题，但不应该包含可以接管账号的内容。

可以保留：

- 时间与时区；
- 使用入口和客户端版本；
- 完整错误信息；
- 是否存在官方禁用通知；
- 最后一个正常步骤；
- 已执行的官方排障步骤。

必须隐藏：

- 密码、会话 Cookie；
- API key、OAuth token、请求头；
- 短信验证码和恢复码；
- 身份证件、人脸和完整付款信息；
- 终端中的环境变量、私有仓库地址或带 token 的 URL。

一张只露出 `403` 的截图通常信息不足；一张把全部请求头贴出来的截图又过度暴露。更合适的是“错误上下文完整，秘密字段全部打码”。

## 一份事实型申诉模板

如果已经确认是个人账号暂停或终止，可以按这个结构整理，而不是复制夸张的“保证解封文案”：

```text
Subject: Request for review of account suspension

Account or organization:
按官方表单要求填写。

Time of incident:
日期、时间、时区。

Message shown:
逐字记录关键提示，不补写猜测。

Context:
说明使用的是网页、Claude Code 还是 API，以及当时进行的正常任务。

Troubleshooting completed:
列出已经执行的官方步骤及结果。

Request:
请复核暂停或终止是否有误，并告知可采取的合规下一步。
```

提交后保存确认信息，不要在短时间内重复发送大量相同工单。如果官方要求补充资料，只在确认的官方渠道提供必要字段。

## 身份验证失败和账号禁用也不要混在一起

Claude 的身份验证页面把“验证失败”和“验证后账号被禁用”分成不同问题：

- 证件或拍摄失败：检查本人有效证件、图像质量和身份验证支持路径；
- 账号被明确禁用：准备事实，进入官方申诉；
- 仅有 `403`：先诊断账号、组织和客户端状态。

买一张新 SIM、换手机号或建立新账号，都不是官方申诉步骤，也不能恢复一个被禁用的账号。
